# 笔记库规则

这个笔记库是一个AI原生笔记系统，让零散的研究想法、会议记录、阅读感悟等可被AI理解并调用。

## 用户信息

- 身份：博士，教授
- 研究领域：经济学金融学、财务理论与资本市场
- 研究方向：数据资产化与资本配置效率、股价崩盘风险
- 主要研究项目：数据资产化与资本配置效率研究

## 目录说明

| 目录 | 内容类型 | 触发场景 |
|------|----------|----------|
| 1_关于我 | 个人身份、目标、背景 | 自我介绍、立场声明 |
| 2_想法 | 研究假设、创新点、构思 | 想法、假设、创新 |
| 3_工作 | 项目任务、待办 | 任务、待办、项目 |
| 4_学习 | 方法笔记、代码技巧 | 方法、技巧、学习 |
| 5_会议 | 讨论记录、组会 | 会议、讨论、组会 |
| 6_研究 | 文献笔记、读后感 | 读文献、文献笔记 |
| 7_行政 | 行政通知、事务 | 通知、事务 |
| 8_附件 | 资料、附件 | 资料、备份 |
| 9_代码 | 代码片段、脚本 | 代码、脚本 |

## 笔记调用规则

### 场景→笔记映射
- **写论文时**：召回2_想法（研究想法）+ 4_学习（方法笔记）
- **读文献时**：召回6_研究（文献笔记）
- **讨论/组会时**：召回5_会议（会议记录）+ 2_想法（想法）
- **分析数据时**：召回4_学习（方法笔记）+ 9_代码（代码）

### 创建笔记规则
1. 用户提到需要记录时，先确认笔记类型
2. 根据内容判断目录归属
3. frontmatter必须包含：note_type, title, created, status, tags

### 检索规则
- 搜索：grep关键词于所有.md文件
- 召回：按场景匹配对应目录的笔记
- 最近：按修改时间排序

## 文件命名

格式：`YYYYMMDD_类型_标题.md`

## 状态

- draft：草稿
- in_progress：进行中
- done：完成
- archived：归档
